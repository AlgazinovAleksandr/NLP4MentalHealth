## Расшифровка

| Колонка | Смысл |
|---------|--------|
| **Schema** | Доля строк, где `answers` проходят проверку кодов, шкал, skip-logic (`questionnaire_validate.py`). |
| **Consistency** | Доля строк, где **и** `expected_total_score`, **и** `expected_flag` совпали с пересчётом по правилам из `registration_questionnaire_v0.0.1.json`. Низкая consistency при высокой schema обычно значит: ответы ок, модель ошиблась в «подписи» к метрике. |
| **Flag OK** | Среди строк с валидной schema: доля, где пересчитанный **флаг** (`relaxed` / `concerned` / `urgent`) совпал с `expected_flag`. |
| **Субъективно /10** | По **качеству данных в строках** (правдоподобность персон, согласованность описания с `answers`, общая аккуратность). **Число строк не штрафуется** — объём смотри в колонке «Строк». Не замена объективным метрикам. |

## Вывод

По **объёму прогона** (50 строк) сильнее **Gemini** и **Qwen**; по **субъективной оценке содержимого** (без учёта N) **GLM** близок к лидерам за счёт **Flag OK** и чистой **schema** на доступной выборке. **gpt-4o-mini** сильнее по **consistency** и **Flag OK** на валидных строках, но даёт **6 битых** записей по схеме. **`expected_total_score` из LLM ненадёжен** у всех — в пайплайне считать балл и флаг кодом. **Mistral** и **GLM** в прогоне дали неполный файл — для датасета фиксированного размера это ограничение; качество отдельных строк см. метрики и **Субъективно**.
